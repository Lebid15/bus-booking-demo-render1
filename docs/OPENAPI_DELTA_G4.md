# تغيرات OpenAPI — Gate G4

الإصدار المنفذ: `4.0.0-e06`.

## المسار المكتمل

- `POST /v1/public/bookings`

## الطلب

يتطلب:

- `Idempotency-Key` بطول 8–120.
- `trip_id` و`hold_token`.
- جهة اتصال باسم وهاتف، وبريد اختياري.
- 1–8 ركاب، وكل راكب مرتبط بـ`seat_id` فريد داخل الطلب.
- طريقة دفع من الطرق المثبتة للرحلة.
- قائمة إصدارات السياسات المقبولة المطابقة تمامًا للإصدارات المثبتة.

## الاستجابة

تعيد:

- `id` وPNR.
- حالة الحجز وحالة الدفع بصورة منفصلة.
- ملخص الرحلة ونقطتي الانطلاق والوصول.
- بيانات الركاب والمقاعد وحالات الصعود المستقلة.
- Snapshot السعر والمهلة وإصدارات السياسات.
- `manage_token` في استجابة الإنشاء فقط.

## أخطاء المجال

- `SEAT_HOLD_EXPIRED`
- `SEAT_HOLD_NOT_OWNED`
- `SEAT_NOT_AVAILABLE`
- `SEAT_GENDER_CONFLICT`
- `PRICE_CHANGED`
- `POLICY_ACCEPTANCE_REQUIRED` — HTTP 428
- `TRIP_NOT_BOOKABLE`

## ضوابط التنفيذ

- لا يثق الخادم بقائمة المقاعد القادمة من الواجهة؛ يعيد تحميل الـHold والمقاعد ويقفلها.
- لا يخزن Manage Token كنص صريح في Booking أو Idempotency.
- توليد OpenAPI والتحقق منه انتهيا بـ0 أخطاء. بقيت خمسة تحذيرات تسمية Enum غير مانعة وموروثة من تعدد حقول `status`.
