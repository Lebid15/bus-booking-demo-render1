# ADR-0001: Modular Monolith

- الحالة: Accepted
- التاريخ: 2026-07-21
- المرجع: `12_ARCHITECTURE/SYSTEM_ARCHITECTURE.md`

## القرار

يُبنى الإصدار الأول كتطبيق Django Modular Monolith بحدود وحدات واضحة، مع Next.js كواجهة مستقلة وPostgreSQL كمصدر الحقيقة. تتواصل الوحدات عبر خدمات تطبيقية وأحداث مجال، وتُرسل الآثار الخارجية عبر Transactional Outbox.

## السبب

دورة الحجز والمقاعد والدفع والدفتر تحتاج معاملات ذرية قوية. تقسيمها المبكر إلى Microservices يزيد مخاطر المعاملات الموزعة والتشغيل دون قيمة تناسب المرحلة الأولى.

## النتائج

- وحدة واحدة قابلة للنشر للـBackend مع حدود تمنع imports العشوائية.
- قابلية فصل الوحدات لاحقًا من خلال عقود الخدمات والأحداث.
- PostgreSQL هو المرجع النهائي للمقاعد والمال، وRedis ليس مصدرًا نهائيًا لهما.
