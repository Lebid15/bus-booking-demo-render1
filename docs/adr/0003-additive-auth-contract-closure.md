# ADR-0003: إضافات غير كاسرة لعقد الهوية

- الحالة: Accepted
- التاريخ: 2026-07-21
- المراجع: `E01-AC01` و`E01-AC05` وترتيب سلطة الوثائق

## السياق

عقد OpenAPI v4.0 يصف تسجيل الدخول وMFA وعرض الجلسات، لكنه لا يدرج Endpoint لإنشاء الحساب أو إلغاء جلسة بعينها، بينما معايير القبول الأعلى تنفيذًا تطلب العمليتين صراحة.

## القرار

أضيفت عمليتان غير كاسرتين:

- `POST /v1/auth/register`
- `DELETE /v1/me/sessions/{session_id}`

لا تُعدل ملفات المواصفة الأصلية. تُسجل الإضافة في Delta التنفيذ وتظهر في المخطط المولد من التطبيق.

## النتائج

تُغلق معايير القبول دون تغيير قاعدة تجارية أو كسر عميل قائم. يجب دمج الإضافتين في النسخة الرسمية التالية من عقد OpenAPI بعد المراجعة.
