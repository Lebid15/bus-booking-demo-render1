# ADR-0002: Access Tokens مرتبطة بجلسة خادمية

- الحالة: Accepted
- التاريخ: 2026-07-21
- المراجع: `06_SECURITY_PRIVACY/AUTH_SESSION_MFA.md` و`E01-AC03`

## القرار

يحمل Access Token معرف المستخدم والجلسة فقط، ويُتحقق في كل طلب من سجل `user_sessions` ومن حالة المستخدم وانتهاء الجلسة وإلغائها. يُخزن Refresh Token كـSHA-256 فقط ولا يُخزن نصه الخام.

## السبب

JWT مستقل لا يحقق الرفض الفوري عند إلغاء الجلسة أو تغيير الدور. الرجوع إلى سجل الجلسة يحقق معيار القبول ويتيح إدارة الأجهزة والجلسات.

## النتائج

- إلغاء الجلسة نافذ فورًا.
- تكلفة قراءة قاعدة بيانات في الطلبات المحمية، ويمكن تخفيفها لاحقًا بذاكرة قصيرة لا تتجاوز متطلبات الإبطال.
- لا تُسجل Tokens في Audit أو Logs.
