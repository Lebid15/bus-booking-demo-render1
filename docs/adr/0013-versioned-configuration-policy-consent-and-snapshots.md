# ADR-0013: الإعدادات الإصداريّة وموافقات السياسات وSnapshot الرحلة

- الحالة: مقبول
- التاريخ: 2026-07-21
- البوابة: G11 / E13

## السياق

تحتاج المنصة والمكاتب إلى إعدادات قابلة للتغيير، لكن تغييرًا حيًا لا يجوز أن يعيد كتابة حقوق حجز أو رحلة قائمة. كما أن السياسات القانونية والتشغيلية تتغير بمرور الوقت، ويجب معرفة الإصدار واللغة التي وافق عليها الزبون دون تخزين بيانات خام غير لازمة. التعديلات الحساسة على مستوى المنصة تحتاج فصل واجبات وتاريخًا قابلًا للتدقيق.

## القرار

1. يوجد Registry مركزي Typed لكل إعداد: النطاق، النوع، الافتراضي، الحدود، الخيارات، الحساسية، وهل يثبت في Snapshot.
2. كل تغيير ينشئ `ConfigurationValue` جديدًا مع `effective_from`؛ لا تعدل القيمة القديمة ولا تحذف.
3. عند نفاذ الإصدار الجديد تُغلق نافذة الإصدار السابق بواسطة `effective_to`.
4. تعديل المكتب يمر عبر سياق المكتب الموثق ويتطلب MFA وسببًا وIdempotency، ويُرفض خارج الحدود.
5. تعديل المنصة ينشأ Pending، ولا يصبح معتمدًا حتى يوافق مستخدم منصة ثانٍ مع MFA؛ يمنع الاعتماد الذاتي.
6. Audit الاعتماد يحتوي before/after والسبب وتاريخ النفاذ وهوية المنشئ والمعتمد.
7. السياسات Immutable versions تحمل وقت النفاذ واللغة والنص وقواعد الآلة وبصمة SHA-256.
8. نشر إصدار مستقبلي لا يغير الرحلات أو الحجوزات السابقة؛ اختيار الإصدار يتم فقط للتدفقات الجديدة بعد وقت النفاذ.
9. عند تأكيد الحجز يجب أن تطابق قائمة الإصدارات المقبولة كل إصدارات Snapshot الرحلة.
10. لكل حجز تنشأ `PolicyAcceptance` لكل إصدار مع subject booking وaccepted_at وhashes فقط لـIP وUser-Agent عند توفرهما.
11. مفاتيح إعداد الرحلة ذات `snapshot=True` تثبت في `Trip.pricing_snapshot.configuration` عند الجدولة وتنتقل إلى Snapshot الحجز.

## النتائج

- لا أثر رجعي لتغيير سياسة أو إعداد.
- يمكن إعادة بناء القيمة النافذة في أي وقت.
- الموافقة قابلة للإثبات حسب الإصدار واللغة والكيان.
- فصل واضح بين اقتراح إعداد منصة واعتماده.
- لا تعتمد قواعد العمل على قيم Frontend ثابتة.

## البدائل المرفوضة

- تحديث صف إعداد واحد in-place: يطمس التاريخ ولا يسمح بإعادة بناء Snapshot.
- اعتبار قائمة IDs المرسلة موافقة دون سجل مستقل: لا يثبت الوقت أو الكيان.
- اعتبار الصمت أو استمرار التدفق موافقة: يخالف شرط الموافقة الصريحة.
- تخزين IP وUser-Agent كنص خام: يجمع بيانات أكثر من اللازم.

## القيود

- النصوص القانونية النهائية تحتاج مراجعة قانونية قبل Production.
- اختبارات الأقفال وقيود PostgreSQL تبقى في بوابة CI المتصلة.
- ربط إعدادات إضافية بالنطاق Branch/Route/Trip سيضاف عند تنفيذ الوحدات التي تستخدمها فعليًا.
