# ADR-0011 — مخزون مقاعد إصداري وإعادة توزيع محكومة وتصعيد دعم قائم على SLA

- الحالة: Accepted
- التاريخ: 2026-07-21
- النطاق: E11

## السياق

تغيير البولمان بعد بيع المقاعد لا يمكن تنفيذه بمجرد استبدال `vehicle_id`. مخطط المقاعد السابق جزء من التذكرة والعقد مع المسافر، وقد توجد مجموعات طفل/مرافق وقواعد جنس وتذاكر مطبوعة. كما أن رفض تذكرة صحيحة أو توقف الرحلة يمثل حقًا تشغيليًا وماليًا يجب ألا يضيع بسبب تعديل صامت أو غياب رد المكتب.

## القرار

1. مخزون الرحلة إصداري؛ كل `TripSeat` يحمل `inventory_version` و`is_current`، ولا يُحذف الإصدار السابق.
2. القيود الفريدة للمقعد تُطبق على المخزون الحالي، بينما يبقى المخزون المؤرشف مرجعًا للتذاكر وسجل التدقيق.
3. تغيير البولمان يمر أولًا عبر `TripReallocationPlan` محاكاة، ولا يُطبق إذا بقي أي `TripReallocationLine` متعارضًا.
4. خوارزمية التوزيع تعطي أولوية للمجموعات المحمية، وتشترط تجاور الطفل والمرافق، وتعيد فحص قاعدة الجنس عبر الحجوزات المستقلة.
5. ترتيب التطبيق ذري: تحرير الـHold، أرشفة المخزون السابق، إنشاء المخزون الجديد، تحويل الإسناد القديم إلى `moved`، إنشاء الإسناد الجديد، ثم إعادة إصدار التذكرة.
6. ينشأ `TripChange` جوهري ورد مستقل لكل حجز؛ يستطيع المسافر القبول أو طلب بديل أو استرداد ضمن مهلة محددة.
7. رفض تذكرة صحيحة يفتح SupportCase P1 تلقائيًا ويجمد المقعد ويرسل Audit وOutbox.
8. حالات الدعم تحمل SLA حسب الأولوية؛ تجاوز مهلة رد المكتب يصعّد الحالة إلى المنصة ويسجل `OfficeViolation` مرة واحدة.
9. أثناء تعطل النظام يسمح تحقق احتياطي محدود بالقائمة أو PNR وهوية جزئية، ولا ينشئ دفعة جديدة.
10. الرحلة المتوقفة تنشئ `TripInterruptionResolution` لكل حجز، ولا يمكن إغلاقها قبل معالجة جميع حقوق الركاب.
11. جميع أوامر HTTP القابلة لإعادة المحاولة تستخدم Idempotency مخزنة ومربوطة بالنطاق والحمولة.

## النتائج

- لا توجد نافذة يصبح فيها الراكب صاحب إسنادين نشطين أو يصبح المقعد مملوكًا لراكبين.
- التذاكر القديمة تبقى قابلة للتدقيق لكنها مبطلة، والتذكرة الجديدة ترتبط بالمقعد الجديد.
- لا يُفصل طفل عن مرافقه ولا تُكسر قاعدة الجنس بصمت؛ التعارض يوقف التطبيق ويطلب معالجة صريحة.
- يستطيع المسافر رفض التغيير أو طلب بديل/استرداد دون فقد أثر القرار.
- حالات P1 والتوقفات التشغيلية لا تختفي بإغلاق الرحلة أو بانتهاء SLA.
- يبقى PostgreSQL 18 بوابة المصدر الحقيقي لاختبارات الأقفال والقيود الجزئية قبل دمج الإصدار إلى `main`.
