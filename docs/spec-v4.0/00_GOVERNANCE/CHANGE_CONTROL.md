# ضبط التغيير

## أنواع التغيير

- **C0 توضيح:** لا يغير السلوك، مثل تحسين صياغة أو مثال.
- **C1 تعديل غير رجعي بسيط:** إضافة حقل اختياري أو تقرير دون تغيير الحقوق.
- **C2 تغيير تشغيلي:** حالة جديدة، مهلة جديدة، أو تغيير صلاحيات.
- **C3 تغيير مالي/قانوني/أمني:** عمولة، استرداد، عزل، بيانات حساسة، أو التزامات تعاقدية.

## متطلبات الطلب

كل طلب تغيير يحتوي: المشكلة، الدافع، البدائل، المتأثرون، أثر البيانات، أثر API، أثر الحالات، أثر المال، خطة الترحيل، خطة الرجوع، واختبارات القبول.

## الموافقات

- C0: مسؤول الوثيقة.
- C1: قائد المنتج + قائد التقنية.
- C2: قائد المنتج + التشغيل + التقنية.
- C3: صاحب المنصة + المالية/القانون/الأمن حسب المجال.

## قواعد الإصدار

- لا تُعدل سياسة نافذة بصمت؛ يُنشأ إصدار جديد بتاريخ نفاذ.
- لا يُحذف حدث مالي أو سجل تدقيق؛ يُعكس أو يُصحح بإدخال جديد.
- لا تُعاد كتابة تاريخ الحجز أو الدفع لإخفاء خطأ.
- تُرفق كل ترقية مخطط بيانات بمسار Roll Forward وRollback آمن.
