# نطاق المرحلة الأولى والنطاق المؤجل

## داخل المرحلة الأولى

- موقع عام متجاوب وحجز كضيف.
- حساب زبون اختياري: الملف والحجوزات فقط.
- لوحة مكتب وفروع وموظفين وصلاحيات.
- بولمانات ومخططات مقاعد ومسارات ورحلات وأسعار.
- مخزون موحد وحجز مؤقت ومنع ازدواج المقاعد.
- مرافقون وقاعدة توافق جنس المقاعد.
- دفع في المكتب وتحويل يدوي؛ واجهة تكامل إلكتروني قابلة للتفعيل.
- PNR وتذكرة وQR وصعود وManifest وNo-show.
- إلغاء واسترداد وتغيير رحلة/مقعد/راكب ضمن السياسات.
- عمولات ودفتر أستاذ وتسويات أساسية.
- دعم ونزاعات ومخالفات وتعليق مكتب.
- إشعارات داخلية وبريد، مع بنية SMS/Push مستقبلية.
- سياسات وشروط وخصوصية وإصدارات موافقة.
- تقارير الرحلات والمبيعات والركاب والمنصة.

## مؤجل

- تطبيق Android أصلي.
- محفظة مالية عامة أو تحويل رصيد بين المستخدمين.
- تذكرة مفتوحة/قسيمة متقدمة.
- حماية إلغاء مدفوعة.
- ولاء ونقاط ومكافآت.
- تتبع GPS مباشر للبولمان.
- API تجاري لوكلاء خارجيين.
- التسعير الديناميكي الشخصي.
- الشحن والطرود والسرافيس والسيارات الخاصة.

## قاعدة منع تضخم النطاق

لا تدخل ميزة مؤجلة إلى التنفيذ إلا بطلب تغيير يوضح القيمة، الأثر المالي، أثر قاعدة البيانات، والدورة التشغيلية والاختبارات.
