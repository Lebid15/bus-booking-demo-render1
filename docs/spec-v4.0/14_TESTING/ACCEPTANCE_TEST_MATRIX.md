# مصفوفة اختبارات القبول النهائية
> **ملحق ملزم:** معايير قبول كل Epic موجودة في `../15_DELIVERY/EPIC_ACCEPTANCE_CATALOG.md` ومصفوفة التتبع في `TRACEABILITY_MATRIX_DETAILED.md`.


## الغرض

هذه الوثيقة هي بوابة القبول التنفيذية للمرحلة الأولى. لا تعتبر أي وحدة مكتملة لمجرد نجاح الواجهة؛ بل يجب إثبات قواعد المجال، العزل، التزامن، المال، التدقيق، والاستعادة على بيئة اختبار تستخدم PostgreSQL حقيقيًا.

## مستويات الدليل

- **AUTOMATED:** اختبار آلي قابل للتكرار ضمن CI.
- **INTEGRATION:** اختبار تكامل مع قاعدة البيانات والمهام الخلفية والتخزين.
- **E2E:** اختبار دورة مستخدم كاملة عبر الواجهة والخادم.
- **MANUAL CONTROLLED:** مراجعة تشغيلية محكومة مع دليل محفوظ.
- **RECOVERY DRILL:** تمرين فشل واستعادة ومصالحة.

## شروط النجاح العامة

1. نجاح جميع اختبارات P0 بلا استثناء.
2. عدم وجود خلل مفتوح من مستوى حرج أو عالٍ في الحجز أو المقاعد أو الدفع أو الصعود أو العزل.
3. عدم وجود دفع أو استرداد أو عمولة غير متوازنة في دفتر الأحداث المالي.
4. حفظ سجل تدقيق لكل تجاوز أو اعتماد مالي أو تغيير حساس.
5. إثبات أن تعديل الإعدادات لا يغير Snapshot حجز سابق.
6. نجاح اختبارات التزامن على مقعد واحد وعدم اعتماد أكثر من تخصيص نهائي.
7. نجاح استعادة نسخة اختبار ومصالحة العمليات المتأخرة قبل اعتماد الجاهزية.

## بوابة الإطلاق التجريبي

لا يبدأ التشغيل الحقيقي حتى:

- نجاح AT-001 حتى AT-035 وفق الأولويات أعلاه.
- توثيق دليل الاختبار ورقم البناء ونسخة قاعدة البيانات.
- إجراء OWNER/OPERATIONS VISUAL REVIEW لتدفقات الزبون والمكتب والمنصة.
- إجراء تمرين انقطاع إنترنت وصعود طوارئ واستعادة قاعدة بيانات.
- اعتماد قائمة المخاطر المقبولة صراحة، ومنع قبول مخاطر تمس ازدواج المقاعد أو المال أو عزل المكاتب.
