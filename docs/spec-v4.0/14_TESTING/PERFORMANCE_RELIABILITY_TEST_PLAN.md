# خطة الأداء والاعتمادية

## سيناريوهات

- بحث رحلات مع آلاف الرحلات النشطة وفلاتر.
- قراءة خريطة مقاعد مع تحديثات متزامنة.
- موجة حجز على رحلة واحدة.
- فتح الصعود ومسح متتابع.
- توليد Manifest وتقارير كبيرة.
- Queue متراكمة بعد عودة مزود.

## قياسات

p50/p95/p99، error rate، DB CPU/IO، lock wait، query count، cache hit، queue lag. توضع ميزانية زمن لكل Endpoint بعد Baseline على بيئة مماثلة.

## Soak

تشغيل ساعات يغطي انتهاء Holds وJobs وWebSockets والتسربات. لا يعتمد النجاح على متوسط فقط؛ تفحص الذروة والأخطاء.
