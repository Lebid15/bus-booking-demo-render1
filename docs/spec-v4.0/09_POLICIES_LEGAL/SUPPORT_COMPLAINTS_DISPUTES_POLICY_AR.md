# سياسة الدعم والشكاوى والنزاعات

> مسودة تشغيلية كاملة للإصدار 1.0. تُراجع قانونيًا وفق القوانين السورية والجهة القضائية المختصة قبل الإطلاق التجاري. لا يجوز استخدام المراجعة القانونية لتغيير الحقوق التشغيلية المتفق عليها دون Change Request.


## القنوات

يقدم الدعم من صفحة الحجز ومركز المساعدة وصندوق المكتب. قد تضاف قنوات أخرى، لكن الحالة داخل النظام هي السجل المرجعي.

## الأولويات

- P0: سلامة أو حادث واسع.
- P1: راكب في الكراج، رفض صعود، رحلة وشيكة، أو دفع يمنع السفر.
- P2: تغيير مؤثر قبل الرحلة.
- P3: استرداد ونزاع مالي غير عاجل.
- P4: استفسار عام.

## الملكية

لكل حالة مالك واضح. لا يطلب من المستخدم التنقل بين المكتب والمنصة دون تسليم رسمي. تحفظ الرسائل والأدلة والقرارات وSLA.

## الأدلة

تشمل الحجز، Snapshot، الدفع، QR، الصعود، الإشعارات، التغييرات، وسجل التدقيق. تعرض أقل بيانات لازمة لكل طرف.

## القرار والاستئناف

يصدر قرار مسبب ويحدد المسؤولية والحل. يسمح باستئناف واحد ضمن المدة عند وجود دليل جديد أو خطأ جوهري. لا ينفذ استرداد مزدوج مع Chargeback.

## الشكاوى ضد المكتب

تؤثر الشكاوى المثبتة في تقييم المخاطر والمخالفات والتسويات ضمن قواعد معلنة، ولا تعتمد على العدد الخام دون فحص الجودة والحجم.


## الإصدارات والموافقة

تنشر الوثيقة برقم إصدار وتاريخ نفاذ ولغة. تحفظ المنصة الإصدار الذي وافق عليه كل مستخدم أو مكتب، ولا تطبق التعديلات الجوهرية بأثر رجعي على حجز قائم.
