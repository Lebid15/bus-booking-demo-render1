# عقد التنفيذ لوكيل البرمجة — v4.0

## التفويض

ينفذ الوكيل المشروع وفق الحزمة، ويحسم التفاصيل التقنية الصغيرة بأفضل الممارسات، لكنه لا يغير نطاق المنتج أو الحقوق أو المال أو الصلاحيات أو الحالات دون Change Request.

## ممنوعات

- لا استخدام Redis كمصدر نهائي للمقاعد.
- لا Float للمال.
- لا ثوابت لعمولة أو مهلة قابلة للإعداد.
- لا تجاوز Tenant scope من معرف يرسله العميل.
- لا حذف Ledger/Audit أو تعديلهما تاريخيًا.
- لا إصدار تذكرة قبل الشروط الموثقة.
- لا اعتبار فشل الإشعار فشلًا للحجز.
- لا اختبارات SQLite للوحدات الحساسة.

## كل PR يجب أن يذكر

Epic، معايير القبول، الجداول والمهاجرات، الحالات، API، الصلاحيات، الأخطاء، الأثر المالي/الخصوصي، الاختبارات، وخطة الرجوع.

## ترتيب التنفيذ

Foundation → Identity → Offices/Fleet → Trips/Inventory → Booking → Payment/Ticket → Boarding/Changes → Finance/Support → Admin/Policies/Reports → Hardening/Pilot.
