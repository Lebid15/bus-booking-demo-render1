# أمر العمل لوكيل البرمجة

1. اقرأ `START_HERE.md` وسلطة الوثائق.
2. لا تنشئ Code قبل ربط المهمة بـEpic ومعيار قبول.
3. نفذ طبقة المجال والقيود قبل الواجهة.
4. لا تستخدم SQLite لاختبارات الحجز والمال؛ استخدم PostgreSQL.
5. لا تضع قيم العمولة أو المهل كثوابت.
6. لا تخفف قيدًا لإمرار اختبار واجهة.
7. كل PR يذكر الجداول والحالات وEndpoints والأخطاء والاختبارات المتأثرة.
8. لا يدمج Epic دون Evidence Bundle.
9. أي غموض يُحسم حسب ترتيب المرجعية؛ إذا كان تغييرًا تجاريًا حقيقيًا ينشأ Change Request ولا يخترع جوابًا.
