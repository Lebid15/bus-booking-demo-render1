# تسليم الإشعارات وإعادة المحاولة

- يسجل `notification_event` ثم يولد Delivery لكل قناة.
- مفتاح dedupe = الحدث + المستلم + القناة + إصدار القالب.
- إعادة المحاولة Exponential Backoff مع حد أقصى وDead Letter Queue.
- الأخطاء الدائمة مثل عنوان غير صالح لا تعاد بلا نهاية.
- لا تؤدي إعادة إرسال إشعار إلى إعادة تنفيذ حدث الحجز أو الدفع.
- تراقب معدلات النجاح والزمن والتأخير لكل مزود.
- عند تعطيل مزود، تبقى الأحداث في Queue أو تحول إلى قناة بديلة حسب حساسية الحدث وموافقة المستخدم.
