# حالات التحميل والفراغ والخطأ والوضع المتدهور

## Loading

- Skeleton يحاكي البنية ولا يعرض بيانات وهمية.
- العمليات المالية تعرض مرحلة واضحة: إرسال الطلب، انتظار التأكيد، المصالحة.
- لا يُغلق Dialog أثناء Mutation إلا بعد نتيجة مؤكدة أو خيار آمن للمتابعة لاحقًا.

## Empty

- يشرح لماذا القائمة فارغة وما الإجراء المنطقي التالي.
- لا يظهر زر إنشاء لمن لا يملك الصلاحية.
- Empty نتائج البحث لا يعني عدم وجود الخدمة؛ يقترح تغيير التاريخ أو النقاط.

## Error

- رسالة للمستخدم + رمز خطأ المجال عند الفائدة + Request ID.
- الأخطاء القابلة لإعادة المحاولة تعرض زرًا واحدًا واضحًا.
- `SEAT_CONFLICT` يعيد المستخدم للخريطة مع تحديثها.
- `VERSION_CONFLICT` يعرض التغييرات الجديدة ويطلب إعادة المراجعة.

## Degraded

- فشل البريد: الحجز مستمر.
- فشل Redis: توقف Holds الجديدة بأمان عند تعذر الضمان؛ الحجوزات النهائية محفوظة.
- فشل مزود الدفع: تخفى القناة مؤقتًا وتبقى القنوات الأخرى.
- فشل قاعدة البيانات: توقف عمليات الكتابة وتظهر حالة النظام؛ لا تستخدم Cache لإنشاء حجز نهائي.
