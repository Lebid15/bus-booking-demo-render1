# قائمة المراجعة البصرية للمالك

## الموقع العام

- هل يفهم المستخدم السعر النهائي وحالة الدفع قبل التأكيد؟
- هل يمكن إكمال الحجز على هاتف 360px دون قص أو تمرير أفقي غير ضروري؟
- هل سياسة الإلغاء ظاهرة قبل الموافقة؟
- هل خريطة المقاعد مفهومة دون الاعتماد على اللون؟
- هل صفحة النجاح تعرض PNR والمهلة والتذكرة بوضوح؟

## لوحة المكتب

- هل يمكن لموظف إنشاء حجز بسرعة من خريطة الرحلة؟
- هل الحجوزات المعلقة والتحويلات والحالات العاجلة ظاهرة في لوحة اليوم؟
- هل لا تُعرض إجراءات مالية لمستخدم غير مخول؟
- هل تغيير البولمان يعرض محاكاة وأثرًا قبل الاعتماد؟

## لوحة المنصة

- هل تميز الواجهة بين المراقبة والتدخل؟
- هل الأثر المالي والتشغيلي ظاهر قبل تعليق مكتب أو إلغاء رحلة؟
- هل سجل التدقيق متاح لكل قرار حساس؟
- هل الفلاتر تحفظ السياق وتدعم التحقيق؟
