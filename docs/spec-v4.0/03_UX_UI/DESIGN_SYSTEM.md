# نظام التصميم

## شخصية الواجهة

هادئة، موثوقة، عملية وسريعة. لا تستخدم زخارف سياحية مبالغًا فيها، ولا تدرجات حادة، ولا ألوانًا تحجب وضوح حالة المقعد أو الدفع.

## الألوان الدلالية

- **Primary:** أزرق/أخضر هادئ للثقة والإجراءات الرئيسية.
- **Success:** للحجز المؤكد والدفع المكتمل والصعود الناجح.
- **Warning:** مهلة دفع، تغيير رحلة، أو إجراء يحتاج انتباهًا.
- **Danger:** إلغاء، فشل دفع، مخالفة، أو إجراء لا يمكن التراجع عنه.
- **Neutral:** الخلفيات والحدود والنصوص الثانوية.

تُحدد القيم النهائية في Tokens، ويجب أن تحقق تباين WCAG AA. لا يعتمد معنى الحالة على اللون وحده؛ يستخدم نص وأيقونة وشارة.

## الخطوط

- خط عربي واضح مثل Tajawal أو Noto Sans Arabic من خلال خدمة/حزمة مرخصة، دون تضمين ملفات خط داخل الحزمة.
- أحجام أساسية: 14 للنص الثانوي، 16 للنص، 20–24 للعناوين، 28–36 للعناوين الرئيسية.
- الأرقام المالية تستخدم محاذاة واضحة وTabular Numerals عند توفرها.

## المكونات الأساسية

1. `AppShell` مع Sidebar قابل للطي ولوحة هاتف Bottom Navigation عند الحاجة.
2. `SmartStatCard` للمؤشرات، دون جعل المؤشر قابلًا للنقر إن لم تكن له وجهة.
3. `TripCard` لنتائج الرحلات: الوقت، المسار، المكتب/الناقل، السعر، المقاعد، الخدمات والسياسة المختصرة.
4. `SeatMap` قابل للوصول بلوحة المفاتيح وقارئ الشاشة.
5. `StatusBadge` موحد للحجز والدفع والرحلة والصعود.
6. `ActionButton` أساسي/ثانوي/خطر، مع Loading وDisabled Reason.
7. `DataCard` للواجهات اليومية، وTable فقط للتقارير الكثيفة.
8. `FilterBar` قابلة للحفظ ومسحها.
9. `Timeline` للتدقيق والحجز والدفع.
10. `PolicySummary` يوضح ما يمكن إلغاؤه ومتى.
11. `MoneyBreakdown` يفصل السعر والرسوم والخصم والمدفوع والمتبقي.
12. `ConfirmationDialog` يعرض الأثر لا سؤالًا غامضًا.

## المقاعد

- المقعد المتاح: حد واضح وخلفية محايدة.
- المحدد: Primary مع رقم واضح.
- غير المتاح: مخفف مع سبب عام “غير متاح وفق سياسة توزيع المقاعد”.
- محجوز مؤقتًا: نمط مميز ومؤقت مرئي للمستخدم صاحب الـHold فقط.
- لا يُكشف جنس أو اسم راكب آخر.
- الأبواب والسائق والممر والدرج والمرافق تمثل كعناصر غير قابلة للحجز.

## RTL/LTR

- العربية RTL بالكامل، مع إبقاء PNR والأرقام الفنية وعناوين البريد LTR داخل عناصر معزولة.
- ترتيب الرحلة منطقي: الانطلاق إلى الوصول، ولا يعكس بصريًا بشكل يغير المعنى.
- الأيقونات الاتجاهية تعكس، لكن أيقونات التشغيل العالمية مثل Play لا تعكس.

## الحركة

- 120–220ms للتغيرات الصغيرة.
- لا تستخدم الحركة كشرط لفهم الحالة.
- احترام `prefers-reduced-motion`.
