# حالة التنفيذ

## Gate G0 — مغلق

- Monorepo وDocker وCI.
- الهوية، الجلسات، MFA، RBAC، سياق المكتب، Audit وOutbox.
- النتائج موثقة في `docs/evidence/G0-foundation/`.

## Gate G1 — مرشح إغلاق

- اعتماد المكاتب والوثائق والفروع والموظفين وحساب التسوية المحمي.
- الجغرافيا والمسارات والأسطول والسائقون ومخططات المقاعد ذات الإصدارات.
- 24 اختبار Backend وفحوص الواجهة نجحت محليًا.
- يبقى نجاح CI على PostgreSQL 18 شرط الدمج النهائي.

## Gate G2 / E04 — مرشح إغلاق

تم تنفيذ:

- قوالب السياسات وإصداراتها غير القابلة للتعديل بعد النشر.
- دورة الرحلة من المسودة حتى الجدولة والنشر وفتح الحجز والصعود والمغادرة والوصول والإكمال والإلغاء.
- فحص جاهزية ذري قبل الجدولة، يشمل المكتب والموارد والوثائق والسياسات والسعر والنوافذ والتعارضات.
- Snapshot للسياسات والأسعار وطرق الدفع والتوقفات ومخطط المقاعد.
- `trip_seats` كأساس ثابت لمخزون الرحلة، و`SeatHold` و`SeatAssignment` بقيود سلامة.
- فتح الحجز تلقائيًا عبر Celery Beat بصورة idempotent.
- تسجيل التغييرات الجوهرية وإنشاء استجابة مطلوبة لكل حجز متأثر.
- حراس المغادرة والإلغاء وإجراءات المعالجة لكل حجز.
- APIs للمكتب وسياسات المنصة والعامة.
- واجهات أولية لرحلات المكتب وسياسات المنصة.

نتائج الجودة المحلية:

- 30/30 اختبار Backend ناجح.
- Django check وmigration drift وRuff وMypy وBandit ناجحة.
- هجرة قاعدة بيانات فارغة ناجحة.
- OpenAPI: 0 أخطاء و5 تحذيرات تسمية Enum موثقة.
- Frontend lint وTypeScript وproduction build ناجحة؛ 11 صفحة و0 ثغرات npm.
- مواصفة v4.0: PASS، مع 119 checksum و68 جدولًا و103 معايير قبول و108 عمليات API و90 شاشة.

قيود الإغلاق النهائي:

1. تشغيل الدورة على PostgreSQL 18 وRedis في CI، وخصوصًا اختبارات تزامن مخزون المقاعد.
2. `pip-audit` المحلي تعذر بسبب DNS نحو PyPI؛ يبقى مفروضًا داخل CI.
3. المعالجة المالية الفعلية لاسترداد إلغاء الرحلة تُستكمل في Epic الدفع/المال، بينما أنشأ E04 إجراءات المعالجة الإلزامية.
