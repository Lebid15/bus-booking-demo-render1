# نشر النسخة التجريبية مجانًا على Render

هذه الحزمة مجهزة لنشر منصة حجز البولمن كبيئة Demo عامة بواسطة ملف `render.yaml`.

## ما الذي ينشأ تلقائيًا؟

- واجهة Next.js عامة.
- Django API عام.
- PostgreSQL 18.
- Redis متوافق عبر Render Key Value.
- تطبيق الهجرات وتهيئة بيانات Demo تلقائيًا.
- تشغيل مهام Celery فوريًا داخل الطلبات في وضع Demo، دون عامل مدفوع منفصل.

## الحسابات التجريبية

| الدور | البريد | كلمة المرور |
|---|---|---|
| مدير المنصة | `admin@demo.local` | `DemoAdmin!2026` |
| مالك المكتب | `office@demo.local` | `DemoOffice!2026` |
| موظف الحجز | `agent@demo.local` | `DemoAgent!2026` |

رمز التحقق التجريبي عند ظهوره: `123456`.

## خطوات النشر

1. ارفع محتويات مجلد المشروع إلى مستودع GitHub خاص أو عام.
2. أنشئ حسابًا مجانيًا في Render وسجّل الدخول.
3. اختر **New > Blueprint** واربط المستودع.
4. تأكد أن Blueprint Path هو `render.yaml` ثم اضغط **Deploy Blueprint**.
5. بعد اكتمال الخدمات، افتح رابط الخدمة `bus-booking-demo-web` من لوحة Render.

لا ترفع ملف `.env.demo` إلى مشروع حقيقي يحتوي بيانات حقيقية. هذه البيئة للتجربة فقط.

## ملاحظات الطبقة المجانية

- قد تحتاج أول زيارة بعد الخمول إلى قرابة دقيقة حتى تستيقظ الخدمات.
- PostgreSQL المجاني على Render مخصص للتجربة ومحدود المدة.
- Redis المجاني غير دائم، لكنه مناسب للجلسات والـrate limits التجريبية.
- لا تستخدم بيانات شخصية أو مالية حقيقية في هذه النسخة.
